# S3 Buckets Deployment Guide

## Overview

SatyaMool uses 3 S3 buckets:
1. **Document Storage** - Encrypted storage for property documents
2. **Audit Logs** - Long-term storage for audit logs
3. **Frontend** - Static website hosting for React application

## Document Storage Bucket

**Bucket Name**: `satyamool-documents-{account-id}`

**Configuration**:
- Encryption: KMS with customer-managed key
- Versioning: Enabled (staging/prod)
- Block Public Access: All enabled
- Lifecycle Policies:
  - Intelligent-Tiering: Immediate transition
  - Delete incomplete multipart uploads: 7 days
  - Delete failed uploads: 7 days (prefix: `failed/`)
  - Delete incomplete verifications: 90 days (prefix: `incomplete/`)

**CORS Configuration**:
```json
{
  "AllowedMethods": ["PUT", "POST"],
  "AllowedOrigins": ["*"],
  "AllowedHeaders": ["*"],
  "MaxAgeSeconds": 3000
}
```

**Bucket Policy**:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": "arn:aws:s3:::satyamool-documents-*/*",
      "Condition": {
        "Bool": {
          "aws:SecureTransport": "false"
        }
      }
    }
  ]
}
```

## Audit Logs Bucket

**Bucket Name**: `satyamool-audit-logs-{account-id}`

**Configuration**:
- Encryption: KMS with customer-managed key
- Versioning: Enabled
- Block Public Access: All enabled
- Lifecycle Policies:
  - Archive to Glacier: 90 days
- Retention: 7 years minimum (compliance requirement)

## Frontend Bucket

**Bucket Name**: `satyamool-{env}-frontend`

**Configuration**:
- Encryption: AES-256 (S3 managed)
- Versioning: Enabled (staging/prod)
- Block Public Access: Disabled for website hosting
- Static Website Hosting: Enabled
  - Index Document: `index.html`
  - Error Document: `index.html` (for SPA routing)

**Bucket Policy** (for CloudFront access):
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::cloudfront:user/CloudFront Origin Access Identity"
      },
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::satyamool-prod-frontend/*"
    }
  ]
}
```

## Security Configuration

### KMS Encryption

All buckets use KMS encryption with automatic key rotation enabled annually.

### Access Control

- Document bucket: Private, accessed via presigned URLs only
- Audit logs bucket: Private, accessed by Lambda functions only
- Frontend bucket: Public read via CloudFront OAI

### Presigned URLs

Generated by API Lambda functions with 15-minute expiration:
```javascript
const presignedUrl = s3.getSignedUrl('putObject', {
  Bucket: documentBucket,
  Key: `properties/${propertyId}/documents/${documentId}`,
  Expires: 900, // 15 minutes
  ContentType: 'application/pdf'
});
```

## Deployment Commands

```bash
# Deploy buckets (included in CDK stack)
cdk deploy

# Verify bucket configuration
aws s3api get-bucket-encryption --bucket satyamool-documents-123456789012
aws s3api get-bucket-versioning --bucket satyamool-documents-123456789012
aws s3api get-bucket-lifecycle-configuration --bucket satyamool-documents-123456789012
```

## Monitoring

Monitor S3 metrics in CloudWatch:
- NumberOfObjects
- BucketSizeBytes
- AllRequests
- 4xxErrors
- 5xxErrors

## Cost Optimization

- Use Intelligent-Tiering for automatic cost optimization
- Archive audit logs to Glacier after 90 days
- Delete incomplete multipart uploads automatically
- Monitor storage usage with CloudWatch alarms
